Game Description

The game is set on a 2D top-down tile-based map where the player builds and manages a transport network. The goal is to create efficient routes, manage vehicles, and generate profit by transporting passengers between stops.
The game runs in real-time and includes adjustable speed settings (pause, normal, fast), which can be changed at any time during gameplay.

Functional Requirements

1. Game World & Map

The game world is a grid-based map consisting of:

Empty tiles → Available for construction
Road tiles → Used for vehicle movement
Stop/Station tiles → Used for passenger pickup and drop-off

Players can:

Build roads
Place stops
Modify and expand the transport network

2. Route & Vehicle System

2.1 Routes

Players can create routes consisting of multiple stops. Each route defines the path that vehicles will follow.
2.2 Vehicles

Each vehicle:

Is assigned to a route
Has a fixed capacity
Moves between stops

2.3 Movement

Vehicles move along roads following their routes with continuous movement.
3. Passenger System

Passengers are generated at stops and have a destination.

Passengers wait at stops
Vehicles pick them up and transport them
Passengers leave the vehicle at their destination

4. Economy System

4.1 Capital & Purchases

Players start with an initial amount of money.
Players can spend money on:

Roads
Stops
Vehicles
Traffic Lights

4.2 Revenue

Players earn money by:

Transporting passengers

More efficient routes result in higher profit.
5. Traffic System

Traffic lights control vehicle movement at intersections.

Vehicles stop at red lights
Vehicles move on green lights
The player can adjust signal timing

6. City Growth

Cities grow over time based on transport activity.

Cities expand to nearby tiles
Growth increases passenger generation

7. Game Speed & Controls

The game has multiple speed levels:

Paused -> Game stops
Normal -> Standard speed
Fast -> Accelerated gameplay

Players can switch speed at any time.
8. Win & Lose Conditions

Win Condition

The player reaches a target amount of money.
Lose Condition

The game ends if the player runs out of money.
Non-Functional Requirements

1. User Interface & Navigation

1.1 Main Menu

The main menu includes:

Start Game
Load Game
Settings
Exit

1.2 In-Game Interface

The interface consists of:

Main map view
Toolbar for actions
Information panel (money, routes, vehicles)
Minimap for navigation

2. Camera & Controls


The player can move the camera across the map
The player interacts using mouse input

3. Performance Requirements


The game should run smoothly (~60 FPS)
Vehicle movement should be efficient
UI interactions should be responsive

4. Technical Requirements


Developed in Java using JavaFX
Object-oriented design
Version control using GitLab

Database Strategy

The game uses a file-based persistence system (JSON format) to store and restore the complete game state. A relational database is not required because the game runs locally and does not require concurrent multi-user access.
Persistence Approach

The system follows a snapshot-based saving strategy:
The entire game state is serialized into a structured JSON file.
On load, the JSON file is deserialized into a GameSnapshot object.
The SaveService is responsible for handling serialization and deserialization.
This approach ensures:

Simplicity of implementation
Human-readable save files (useful for debugging)
Easy portability across systems
Stored Data Structure
The saved JSON file contains the following structured data:
1. Map Data

Map width and height
Tile types (road, empty, stop, etc.)
Stop positions
Traffic light positions and states
2. Routes

Route ID
Route name
Ordered list of stop IDs
3. Vehicles

Vehicle ID
Vehicle type (Bus, Tram, Taxi)
Assigned route ID
Current position (GridPos)
Current state (Idle, Moving, Boarding)
Speed
4. Passengers

Passenger ID
Origin stop ID
Destination stop ID
Current status (waiting / onboard)
5. Economy Data

Player balance
Revenue history (optional extension)
Expenses (optional extension)
6. Game Time

Current game time (milliseconds or ticks)
Current game speed
Current game state (Running / Paused / Ended)
